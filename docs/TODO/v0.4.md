# v0.4 — product features

Goal: make the video itself noticeably better and the project plausibly
shareable. v0.3 is about not breaking; v0.4 is about being worth using.

## headline features (pick one or two, not all)

- [ ] **Elevation profile overlay (Relive-style HUD).** Bottom strip of
      the frame shows the GPX elevation curve with a moving cursor at
      the current position; current km / current ele numbers next to
      it. This is the single most identifying visual feature of the
      genre and we don't have it yet. Implementation: SVG layer in
      `public/index.html` driven by `setFrame()`.
- [ ] **Map style switcher.** Today the satellite + 3D-terrain look is
      hardcoded to MapTiler. Abstract a "style provider" interface so a
      user can `--style satellite|topo|outdoors|custom_url` from CLI
      and pick from the UI. Lets people render hikes on a topo map and
      city runs on a vector style.
- [ ] **Two GPX overlay (this year vs last year, me vs friend).**
      Different colors per track. Camera path picks one as the primary
      flyover, second renders as a static line behind. The UI accepts
      a second file slot.

## supporting features

- [ ] **Background music input.** `--audio path/to/track.mp3` mixes audio
      into the final mp4 via ffmpeg. UI accepts an audio drop too.
- [ ] **WebM and GIF outputs.** Smaller share-friendly artifacts. WebM
      via libvpx; GIF via the existing frames pipeline + gifski.
- [ ] **Title overlay typography pulled from the topographic UI.** Reuse
      Fraunces in the rendered video text — currently the `<style>` in
      `public/index.html` is its own hand-rolled font stack.

## release-blocking polish (for npm publish, deferred from v0.2)

- [ ] **`bin: trail-render` entry in package.json + shebang in src/index.js.**
      Lets users `bunx trail-render hike.gpx` without cloning the repo.
- [ ] **`--doctor` subcommand.** Probes ffmpeg, chromium, MAPTILER_KEY,
      reports what's missing. Saves a round-trip on first install.
- [ ] **README `What it looks like` section gets a real GIF.** Render a
      10-second clip from the demo GPX and embed at the top of README.md.
      A static text bullet list isn't selling the project.
